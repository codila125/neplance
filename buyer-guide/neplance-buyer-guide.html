What This App Does

Neplance is similar in idea to platforms like Upwork or Fiverr, but built as a custom product. It helps two groups meet in one place: people who need work done and freelancers who can do that work.

The app is not only a landing page or design concept. It already includes the main marketplace flow: account creation, job posting, proposals, contract creation, internal payment tracking, messaging, notifications, reviews, and admin tools.

Who uses it

  • Clients: post jobs, receive proposals, hire freelancers, manage payments
  • Freelancers: browse jobs, apply, complete work, collect earnings
  • Admins: review disputes and verification requests

Business value

  • Creates trust between both sides
  • Keeps the full work process in one system
  • Can be expanded into a production marketplace business

How The Platform Works In Simple Words

  1. Step 1
    A user signs up and chooses whether they are acting as a client or a freelancer.
  2. Step 2
    A client posts a job with title, description, budget, deadline, and required skills.
  3. Step 3
    Freelancers browse available jobs and send proposals with price, message, and delivery time.
  4. Step 4
    The client reviews proposals and turns one of them into a contract.
  5. Step 5
    The contract is funded inside the app, the freelancer signs it, and the work begins.
  6. Step 6
    Both sides can chat, track progress, approve delivery, and finish the contract.
  7. Step 7
    The app updates balances, creates notifications, stores reviews, and allows dispute handling if needed.

What Is Already Built

User accounts and roles

Login, signup, protected dashboards, and role-based access are in place. One account can act as more than one role.

Job marketplace

Clients can create and publish jobs. Freelancers can browse jobs and view detailed job pages.

Proposal system

Freelancers can submit proposals and clients can review, accept, or reject them.

Contracts and work tracking

The system supports contract creation, signing, delivery, change requests, completion, and cancellation.

Internal wallet and escrow logic

The app tracks available balance, held funds, releases, refunds, and earnings inside a wallet ledger.

Messaging and notifications

Users can communicate within the platform and receive activity alerts for proposals, contracts, disputes, and chat messages.

Trust and safety features

User verification, reviews, and admin dispute resolution are part of the current project.

Admin area

Admin users can review pending verification requests, disputes, jobs, and user-related information.

Technology, Explained Simply

Frontend

This is the part users see in the browser. It is built with Next.js and React, which are common modern tools for making fast web apps.

Backend

This is the logic behind the scenes. It manages users, jobs, contracts, wallets, permissions, and business rules using Express.

Database

Data is stored in MongoDB. That means users, jobs, proposals, and financial records can be saved and retrieved reliably.

Security

The app already includes login protection, role-based permissions, tokens, cookies, and guarded routes for authenticated areas.

What Makes This Project Valuable

  • It separates the visual app and the server logic cleanly, which makes future upgrades easier.
  • It already includes important trust features like verification, reviews, and dispute handling.
  • It is suitable as a base product for a startup, agency offering, or custom freelance platform.

Important Honest Notes For The Buyer

  • The wallet system works as an internal ledger. It is not yet a real live payment gateway integration.
  • Blockchain-related fields and files exist, but the core working payment flow is currently the internal wallet logic.
  • Some marketing numbers and testimonials on the landing page appear to be sample content for presentation.
  • One admin page is still a placeholder, so not every admin screen is fully finished.
  • The project is a strong functional base, but a buyer may still want final polishing, real payment integration, and deployment hardening before large public launch.
